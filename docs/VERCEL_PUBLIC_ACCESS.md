# Vercelでアプリを公開する方法

Vercelのログイン画面に遷移してしまう場合の対処法です。

## 問題の原因

Vercelのプロジェクトがプライベート設定になっている、またはデプロイメントが正常に完了していない可能性があります。

## 解決方法

### 1. デプロイメントの状態を確認

1. [Vercel Dashboard](https://vercel.com/dashboard)にアクセス
2. `quill` プロジェクトを開く
3. 「Deployments」タブを確認
4. 最新のデプロイメントが「Ready」状態になっているか確認

### 2. デプロイメントURLを確認

1. Vercelダッシュボードでプロジェクトを開く
2. 「Deployments」タブで最新のデプロイメントをクリック
3. 「Visit」ボタンをクリックして、正しいURLを確認
4. URLの形式: `https://quill-xxxxx.vercel.app` または `https://quill.vercel.app`

### 3. デプロイメントの種類を確認

1. Vercelダッシュボードでプロジェクトを開く
2. 「Deployments」タブで最新のデプロイメントを確認
3. デプロイメントの左側に「Production」または「Preview」のバッジが表示されているか確認

### 4. 本番環境のURLを確認

1. Vercelダッシュボードでプロジェクトを開く
2. プロジェクト名の下、または「Overview」タブに表示されるURLを確認
3. 通常は以下のいずれかの形式：
   - `https://quill.vercel.app`（プロジェクト名のみ）
   - `https://quill-xxxxx.vercel.app`（自動生成されたURL）

### 5. プレビューデプロイメントを本番環境にプロモート（必要な場合）

「Promote to Production」ボタンが見つからない場合、以下の方法を試してください：

**方法1: デプロイメントのメニューから**
1. 「Deployments」タブで最新のデプロイメントをクリック
2. デプロイメント詳細画面の右上の「...」（三点メニュー）をクリック
3. 「Promote to Production」を選択

**方法2: デプロイメント一覧から**
1. 「Deployments」タブで最新のデプロイメントにマウスオーバー
2. 表示される「...」メニューをクリック
3. 「Promote to Production」を選択

**方法3: 既に本番環境の場合**
- デプロイメントが既に「Production」として表示されている場合、プロモートは不要です
- プロジェクトの本番URL（`https://quill.vercel.app`など）に直接アクセスしてください

### 6. プロジェクトの設定を確認

1. Vercelダッシュボードでプロジェクトを開く
2. 「Settings」>「General」を選択
3. 「Project Name」を確認（通常は `quill`）
4. プロジェクト名の下に表示される本番URLを確認
5. このURLにアクセスしてアプリが表示されるか確認

### 7. カスタムドメインの設定（オプション）

もしカスタムドメインを設定している場合：
1. 「Settings」>「Domains」を確認
2. 正しいドメインが設定されているか確認

### 8. 環境変数の確認

1. 「Settings」>「Environment Variables」を確認
2. 以下の環境変数が全て設定されているか確認：
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   - `VITE_GEMINI_API_KEY`
   - `VITE_AUTH_PASSWORD`
3. **重要**: 全ての環境（Production, Preview, Development）に設定されているか確認

### 9. ビルドログの確認

1. 「Deployments」タブで最新のデプロイメントをクリック
2. 「Build Logs」を確認
3. エラーがないか確認
4. エラーがある場合は、エラーメッセージを確認して修正

### 10. 再デプロイ

問題が解決しない場合：
1. GitHub Actionsを再実行
   - GitHubリポジトリの「Actions」タブ
   - 最新のワークフローを選択
   - 「Re-run jobs」をクリック
2. または、Vercelダッシュボードから再デプロイ
   - 「Deployments」タブ
   - 最新のデプロイメントの「...」メニューから「Redeploy」

## 正しいアクセス方法

1. Vercelダッシュボードでプロジェクトを開く
2. 「Deployments」タブで最新のデプロイメントを確認
3. 「Visit」ボタンをクリック
4. または、URLを直接コピーしてブラウザで開く

## トラブルシューティング

### ログイン画面が表示されない

- ブラウザのコンソール（F12）でエラーを確認
- 環境変数が正しく設定されているか確認
- ビルドログでエラーがないか確認

### 白い画面が表示される

- ブラウザのコンソールでエラーを確認
- ネットワークタブでリソースの読み込みエラーを確認
- 環境変数が正しく設定されているか確認

### 404エラーが表示される

- Vercelの設定でルーティングが正しく設定されているか確認
- `vercel.json`の設定を確認

## 確認チェックリスト

- [ ] デプロイメントが「Ready」状態
- [ ] プロジェクトが「Public」設定
- [ ] 環境変数が全て設定されている
- [ ] ビルドログにエラーがない
- [ ] 正しいURLにアクセスしている
- [ ] ブラウザのコンソールにエラーがない
