# Quill 仕様書

## プロジェクト概要

**プロジェクト名**: Quill  
**説明**: クレジットカードの利用履歴を閲覧・管理するためのWebアプリケーション  
**リポジトリ名**: quill

## インフラストラクチャ

### ホスティング
- **フロントエンド**: Vercel（無料枠）
- **バックエンド**: Supabase（無料枠）
  - データベース: PostgreSQL（500MB無料枠）
  - 認証: アプリ側で実装（Supabase認証は使用しない）

### 外部API
- **Google Gemini API**: AIによる自動カテゴリ分類（無料枠使用）
  - 月60リクエスト/分、1日1,500リクエストまで無料

## 認証・アクセス制御

### 認証方式
- **シンプルなワード認証**
  - 指定したパスワード（環境変数 `VITE_AUTH_PASSWORD`）でログイン
  - メールアドレスや複雑な認証は不要

### ロール管理
- **管理者（admin）**
  - ユーザー名: `admin`
  - 権限:
    - データの閲覧
    - CSVファイルのアップロード
    - 個別取引の非表示設定
    - デフォルト非表示設定の管理
    - カテゴリマッピングの管理（AI分類結果の調整）

- **閲覧者（viewer）**
  - ユーザー名: `viewer`
  - 権限:
    - データの閲覧のみ
    - 非表示設定された取引は表示されない

## データ構造

### 取引データ（transactions テーブル）
- 取引日時
- 出金金額（円）
- 入金金額（円）
- 海外出金金額
- 通貨
- 変換レート（円）
- 利用国
- 取引内容（支払い、チャージ、投資など）
- 取引先（店舗名）
- 取引方法（PayPay残高、PayPayカード、クレジットカードなど）
- 支払い区分
- 利用者
- 取引番号（ユニークキー）
- カテゴリ（AI分類結果）
- 非表示フラグ

### カテゴリマッピング（category_mappings テーブル）
- 取引先名（ユニークキー）
- カテゴリ名
- 作成日時
- 更新日時

### デフォルト非表示設定（default_hidden_settings テーブル）
- 設定タイプ（取引方法 / 取引内容）
- 設定値
- 作成日時
- 更新日時

## 機能仕様

### 1. CSVアップロード機能
- **対象ユーザー**: 管理者のみ
- **対応フォーマット**: PayPayからエクスポートしたCSVファイル
- **処理フロー**:
  1. CSVファイルをアップロード
  2. CSVをパースして取引データに変換
  3. ユニークな取引先名を抽出
  4. 既存のカテゴリマッピングを確認
  5. 未分類の取引先をGoogle Gemini APIで分類
  6. カテゴリマッピングを保存
  7. デフォルト非表示設定を適用
  8. 取引データをデータベースに保存

### 2. データ閲覧機能
- **表示形式**: PayPayアプリ風のリスト形式
- **グループ化**: 月別にグループ化して表示
- **表示項目**:
  - 取引先アイコン（簡易表示）
  - 取引先名
  - 取引日時（「2026年3月7日 10時13分」形式）
  - 取引方法（アイコン付き）
  - ステータスバッジ（支払い完了、付与処理中など）
  - 金額（出金は黒、入金は緑、ポイントは緑で「pt」表示）

### 3. 検索・フィルター機能
- **テキスト検索**: 取引先名、取引内容で検索
- **フィルター条件**:
  - 日付範囲（開始日〜終了日）
  - 金額範囲（最小〜最大）
  - 取引方法（PayPay残高、PayPayカードなど）
  - 取引内容（支払い、チャージ、投資など）
  - カテゴリ（外食、コンビニ、ショッピングなど）
- **フィルターUI**: 
  - 上部にフィルターバー（クイックフィルター）
  - 詳細フィルターパネル（モーダル）

### 4. 非表示設定機能

#### 4.1 個別取引の非表示
- **対象ユーザー**: 管理者のみ
- **機能**: 各取引に「非表示」ボタンを表示
- **動作**: 非表示にした取引は一覧に表示されない（管理者は設定可能、閲覧者は非表示の取引を見られない）

#### 4.2 デフォルト非表示設定
- **対象ユーザー**: 管理者のみ
- **設定項目**:
  - 取引方法ごとの非表示設定（例：「PayPayポイント」を非表示）
  - 取引内容ごとの非表示設定（例：「投資」を非表示）
- **動作**: 
  - CSVアップロード時に自動適用
  - 既存データにも反映可能（今後実装予定）

### 5. 利用レポート機能
- **表示内容**:
  - 出金合計金額
  - カテゴリ別集計（円グラフ表示）
  - 取引先ランキング（上位10店舗）
  - 月別推移（棒グラフ表示）
- **データ範囲**: フィルターで絞り込んだデータを集計

### 6. AIカテゴリ分類機能
- **使用API**: Google Gemini API（gemini-proモデル）
- **分類カテゴリ**:
  - 外食
  - コンビニ
  - スーパー
  - ショッピング
  - ガソリン
  - 医療
  - 交通
  - 娯楽
  - 投資
  - 公共料金
  - その他
- **処理方法**:
  - バッチ処理で複数の取引先を一度に分類
  - 既に分類済みの取引先はスキップ
  - 分類結果をデータベースに保存して再利用

## 環境変数

```env
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_GEMINI_API_KEY=your_gemini_api_key
VITE_AUTH_PASSWORD=your_auth_password
```

## データ保持期間

- Supabase無料枠: 500MB
- 現在のCSVファイル: 約41KB（2ヶ月分）
- 計算上、約2,000年分のデータを保持可能（実用上は数十年分は問題なし）
